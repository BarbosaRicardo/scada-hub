\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{iblue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{Introduction} --- Before IEC 61131-3, every PLC manufacturer used a proprietary language. Engineers retrained from scratch when they changed vendors. This chapter explains why the standard exists, what it unified, and which of the five languages to reach for first.
  \item \textbf{Data Types} --- BOOL, INT, REAL, TIME, STRING, and user-defined structured types. The type system is what separates safe PLC code from implicit conversions that silently corrupt process values at runtime.
  \item \textbf{Structured Text (ST)} --- Pascal-style text language: IF/THEN, FOR, WHILE, CASE, function calls. The most expressive of the five languages and the most transferable skill for engineers coming from software backgrounds.
  \item \textbf{Ladder Diagram (LD)} --- Relay-logic graphical representation: contacts, coils, and rungs. Electricians can read it without training. Every PLC vendor supports it. Understanding LD is non-optional for anyone working with existing automation code.
  \item \textbf{Function Block Diagram (FBD)} --- Signal-flow graphical language where data passes between interconnected function blocks. The natural fit for control engineers working with PID loops, signal conditioning, and process instrumentation.
  \item \textbf{Sequential Function Chart (SFC)} --- State machine language for batch and sequential processes. Steps, transitions, and divergence structures replace thousands of lines of LD timer-and-flag logic with a readable, maintainable flowchart.
  \item \textbf{Program Organization Units} --- Functions, Function Blocks, Programs, and Configurations are the module system that enables reusable automation libraries. Engineers learn how to build code that works across vendors without modification.
  \item \textbf{RTAC Integration} --- Applying IEC 61131-3 logic in the SEL RTAC; how the standard crosses from PLC programming into substation protection and automation. The bridge between this course and the RTAC guide.
  \item \textbf{Troubleshooting} --- Type mismatches, scan cycle overruns, unintended coil latching, and the runtime faults that crash programs without a clear error message. Field debugging procedures for real PLC hardware.
  \item \textbf{Practice Lab} --- Four exercises: a PID temperature controller in Structured Text, a motor starter sequence in Ladder, a batch filling process in SFC, and a reusable Function Block library for analog scaling.
\end{enumerate}
