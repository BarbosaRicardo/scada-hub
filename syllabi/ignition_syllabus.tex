\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{oblue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{Gateway Architecture:} Traditional SCADA platforms charged per tag and per client seat. Ignition's unlimited licensing model changed the economics of the entire market. This chapter covers Gateway installation, the module system, the web management interface, and the backup and restore procedures that protect a running production system.
  \item \textbf{Tag System:} Every piece of data in Ignition is a tag. OPC tags read from PLCs. Expression tags compute derived values. Memory tags hold state. UDTs create typed, reusable templates for equipment that appears dozens of times on a single plant floor. The tag system is the foundation everything else builds on.
  \item \textbf{OPC-UA Integration:} Ignition connects to Siemens, Allen-Bradley, Modbus, and any OPC-UA compliant device through a single protocol. This chapter covers Ignition as both OPC-UA server and client, and the device connection configuration that takes a new PLC from unconfigured to live tags in under an hour.
  \item \textbf{Perspective Module:} HMI running in any browser, on any device, without plugins. This chapter covers the component library, View/Page/Session architecture, and the binding system that connects every displayed value to a live tag without writing code for the common case.
  \item \textbf{Scripting:} Python (Jython) scripting runs on tag events, gateway events, a schedule, and in response to user actions. This chapter covers the system library that gives scripts access to tags, databases, alarm pipelines, and messaging. Automation logic that runs without a PLC compile cycle.
  \item \textbf{Alarm Notification:} An alarm that reaches no one is equivalent to no alarm. This chapter covers alarm modes, notification pipelines, escalation rules, roster configuration, and email and SMS delivery. The infrastructure that ensures a fault reaches the right person at 2~a.m.
  \item \textbf{Tag Historian:} The historian stores process data for trend analysis, regulatory compliance, and root cause investigation. This chapter covers storage providers, partitioning strategy, aggregation modes, and Store and Forward, the feature that preserves data through network interruptions without gaps in the historical record.
  \item \textbf{Database Integration:} Named queries connect Ignition to any SQL database without embedding SQL in scripts. Transaction Groups push tag data to database tables on scan or on change. This chapter closes the loop from field sensor to management report.
  \item \textbf{Security and Access Control:} Identity providers, role mapping, project-level permissions, and SSL/TLS configuration. The Ignition security model scales from a single-site deployment with five users to a multi-site enterprise with hundreds of engineers and operators.
  \item \textbf{Field Lab:} Build a complete SCADA application from scratch: PLC connection via OPC-UA, tag configuration, Perspective dashboard with live trends and alarm displays, alarm notification pipeline, historian storage, and a management report generated from historical data.
\end{enumerate}
