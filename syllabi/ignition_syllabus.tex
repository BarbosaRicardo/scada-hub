{\small\sffamily\color{ogray}\MakeUppercase{Course Description}}
\smallskip

{\small Ignition is a web-deployed SCADA platform built on OPC UA, used across manufacturing, water/wastewater, and energy sectors. This course covers the gateway architecture, tag system design, Perspective HMI development, Jython scripting, alarm notification pipelines, Tag Historian, database integration, and security configuration — building a complete SCADA application from device connection to management report.}

\smallskip
{\small\sffamily\color{ogray}\MakeUppercase{Prerequisites}}
\smallskip

{\small Basic SQL knowledge is helpful. No prior SCADA or HMI experience required.}

\smallskip
{\color{onavy}\rule{\linewidth}{1.2pt}}
\smallskip
{\small\sffamily\color{ogray}\MakeUppercase{Course Topics}}
\smallskip

\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{oblue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{Gateway Architecture:} Installation, module system, web management interface, unlimited licensing model, and backup/restore procedures.
  \item \textbf{Tag System:} OPC, expression, and memory tag types; UDT templates for repeated equipment; tag database as the system foundation.
  \item \textbf{OPC-UA Integration:} Ignition as OPC-UA client and server, device driver configuration, and PLC-to-live-tags workflow.
  \item \textbf{Perspective Module:} Browser-based HMI architecture, component library, View/Page/Session structure, and tag binding without custom code.
  \item \textbf{Scripting:} Jython on tag events, gateway events, and schedules; system library access to tags, databases, and alarm pipelines.
  \item \textbf{Alarm Notification:} Alarm modes, notification pipelines, escalation rules, roster configuration, and email/SMS delivery infrastructure.
  \item \textbf{Tag Historian:} Storage providers, partitioning strategy, aggregation modes, and Store and Forward for gap-free historical data.
  \item \textbf{Database Integration:} Named queries, Transaction Groups, and the path from field sensor value to SQL table row.
  \item \textbf{Security and Access Control:} Identity providers, role mapping, project permissions, and SSL/TLS configuration across single-site and multi-site deployments.
  \item \textbf{Field Lab:} End-to-end SCADA build — OPC-UA device connection, tag configuration, Perspective dashboard, alarm pipeline, historian, management report.
\end{enumerate}

\smallskip
{\color{onavy}\rule{\linewidth}{0.4pt}}
\smallskip
{\small\sffamily\color{ogray}\MakeUppercase{Learning Outcomes}}
\smallskip

\setlist[itemize]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{oblue}{\small$\bullet$}}
\begin{itemize}\small
  \item Connect an OPC-UA device and expose live tags in a functioning Ignition project.
  \item Build a Perspective dashboard with live trend displays, alarm status, and navigation.
  \item Configure an alarm notification pipeline with escalation rules and SMS/email delivery.
  \item Write Jython scripts for tag event handling, scheduled tasks, and database queries.
  \item Query historian data and generate a management report from stored process values.
\end{itemize}
