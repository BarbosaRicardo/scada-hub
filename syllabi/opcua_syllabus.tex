{\small\sffamily\color{ugray}\MakeUppercase{Course Description}}
\smallskip

{\small OPC UA is the successor to the classic OPC specifications and the standard communication backbone for Industry 4.0 and IIoT architectures. This course covers the client/server and pub/sub transport models, the node-based address space, the full service set, session and SecureChannel mechanics, and the certificate-based security model. Integration patterns using open-source stacks and commercial platforms are examined alongside Wireshark-based diagnostic techniques.}

\smallskip
{\small\sffamily\color{ugray}\MakeUppercase{Prerequisites}}
\smallskip

{\small Basic TCP/IP networking concepts. Completion of the Modbus course is recommended but not required.}

\smallskip
{\color{unavy}\rule{\linewidth}{1.2pt}}
\smallskip
{\small\sffamily\color{ugray}\MakeUppercase{Course Topics}}
\smallskip

\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{ublue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{OPC UA Architecture:} Client/server and pub/sub transport models, the transition from DCOM, and the platform-neutral unified namespace.
  \item \textbf{Information Model:} Address space, node classes, NodeIDs, type definitions, and the self-describing browsing model.
  \item \textbf{Services:} Read, Write, Browse, Subscribe, and CreateMonitoredItems — the full service set and which matter in practice.
  \item \textbf{Sessions and Channels:} SecureChannel negotiation, session establishment, token lifetime, and reconnection behavior on channel loss.
  \item \textbf{Security:} Security Mode None vs.\ Sign vs.\ SignAndEncrypt; certificate exchange; PKI structure; consequences of disabled security.
  \item \textbf{Subscriptions:} Publishing interval, MonitoredItem sampling, deadband filters, and the configuration chain controlling network traffic volume.
  \item \textbf{Data Types:} Scalars, structures, enumerations, arrays, and the StatusCode quality system for representing sensor validity.
  \item \textbf{Discovery:} Local and Global Discovery Servers, endpoint enumeration, certificate trust, and open-discovery security implications.
  \item \textbf{Implementation:} SDK selection, Ignition as OPC UA server, common integration failures, and Wireshark-based diagnostic workflow.
  \item \textbf{Protocol Lab:} Full session capture — TCP connection, SecureChannel, browse, subscription, disconnect, and session token renewal.
\end{enumerate}

\smallskip
{\color{unavy}\rule{\linewidth}{0.4pt}}
\smallskip
{\small\sffamily\color{ugray}\MakeUppercase{Learning Outcomes}}
\smallskip

\setlist[itemize]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{ublue}{\small$\bullet$}}
\begin{itemize}\small
  \item Browse an OPC UA server's address space and interpret node classes and references.
  \item Configure Security Mode Sign and SignAndEncrypt with certificate exchange on a production server.
  \item Build and tune a subscription with appropriate publishing intervals and deadband filters.
  \item Verify security mode and session token behavior from a Wireshark packet capture.
  \item Select and configure an OPC UA SDK or integration platform for a given deployment scenario.
\end{itemize}
