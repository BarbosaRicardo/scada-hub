\smallskip

\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{rgreen}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{RTAC Architecture} --- The SEL RTAC (SEL-3505, SEL-3530, SEL-3555) is a protocol gateway, data concentrator, and local logic engine in one ruggedized DIN-rail device. This chapter covers hardware variants, I/O modules, communication ports, and the commissioning sequence before any software is opened.
  \item \textbf{ACSELERATOR Architect} --- SEL's configuration software organizes every RTAC project as a database: tags, aliases, point mapping, and revision control. Engineers learn the project structure that every RTAC configuration follows and the workflow that prevents configuration drift.
  \item \textbf{DNP3 Configuration} --- The RTAC communicates upstream to the SCADA master via DNP3. This chapter covers master and outstation roles, point database mapping, event handling, unsolicited reporting, and the DNP3 parameters that determine how quickly a SCADA system sees a field change.
  \item \textbf{Modbus Gateway} --- The RTAC translates Modbus TCP requests into DNP3 point values. This chapter covers server configuration, register mapping from DNP3 to coils and holding registers, and the common integration patterns for legacy SCADA systems that speak only Modbus.
  \item \textbf{IEC 61850 Integration} --- GOOSE messages carry protection relay trip signals across the station LAN in microseconds. Sampled Values carry current and voltage waveforms. MMS handles operator control. This chapter covers all three and how RTAC maps IEC 61850 data to its internal tag database.
  \item \textbf{Local Logic} --- When communications to the SCADA master fail, the RTAC continues executing local IEC 61131-3 programs. This chapter covers the ACSELERATOR logic engine: event-driven automation, failover sequences, and the programs that keep a substation safe when the network is down.
  \item \textbf{Security \& Access Control} --- Role-based access, RBAC profile configuration, serial port security, and firmware update procedures. The RTAC is a critical infrastructure device; this chapter covers the security posture required for NERC CIP compliance in power utility applications.
  \item \textbf{Field Scenarios} --- Three substation automation case studies: transformer bay control with automatic tap changer response, feeder protection coordination with downstream reclosers, and load shedding automation that executes correctly under a communication fault to the SCADA master.
\end{enumerate}
