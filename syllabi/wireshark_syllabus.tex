{\small\sffamily\color{wgray}\MakeUppercase{Course Description}}
\smallskip

{\small Wireshark is the primary tool for ICS protocol analysis, incident investigation, and security verification in operational technology networks. This course covers capture techniques using SPAN ports and network taps, display filter construction, and dissector-level analysis of Modbus, DNP3, and OPC UA traffic. Advanced topics include tshark scripting, remote capture over SSH, and Lua dissectors for proprietary protocols.}

\smallskip
{\small\sffamily\color{wgray}\MakeUppercase{Prerequisites}}
\smallskip

{\small Completion of at least one protocol course (Modbus, OPC UA, or DNP3) is strongly recommended.}

\smallskip
{\color{wnavy}\rule{\linewidth}{1.2pt}}
\smallskip
{\small\sffamily\color{wgray}\MakeUppercase{Course Topics}}
\smallskip

\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{wblue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{Introduction to Wireshark:} Three-pane interface, dissector architecture, ICS protocol support, and the difference between tshark and dumpcap.
  \item \textbf{Capture Techniques:} SPAN ports, network taps, promiscuous mode, ring buffers, pcap/pcapng formats, and remote capture over SSH.
  \item \textbf{Display Filters:} BPF capture filters, Wireshark display filter syntax, SCADA-specific expressions, colorization rules, and Follow TCP Stream.
  \item \textbf{Protocol Dissectors:} Dissector mechanics, Decode As for non-standard ports, Wireshark profiles, and Statistics tools for traffic pattern analysis.
  \item \textbf{Modbus Analysis:} MBAP header fields, transaction ID tracking, write command identification, and exception response detection in captures.
  \item \textbf{DNP3 Analysis:} Frame structure, unsolicited response identification, IIN bit monitoring, and Secure Authentication v5 handshake verification.
  \item \textbf{OPC UA Analysis:} Session establishment, SecureChannel negotiation, security mode verification, and subscription traffic profiling from captures.
  \item \textbf{Security Analysis:} Traffic baseline establishment, anomaly detection, scanning identification, and incident response capture procedures.
  \item \textbf{Advanced Techniques:} tshark scripting, SSH remote capture, Lua dissectors for proprietary protocols, and packet replay with tcpreplay.
  \item \textbf{Protocol Lab:} Modbus poll overload diagnosis, Exception Code 02 trace, OPC UA security audit, rogue master detection, tshark alert script.
\end{enumerate}

\smallskip
{\color{wnavy}\rule{\linewidth}{0.4pt}}
\smallskip
{\small\sffamily\color{wgray}\MakeUppercase{Learning Outcomes}}
\smallskip

\setlist[itemize]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{wblue}{\small$\bullet$}}
\begin{itemize}\small
  \item Capture ICS protocol traffic using SPAN ports and network taps without network disruption.
  \item Write Wireshark display filters targeting Modbus, DNP3, and OPC UA traffic patterns.
  \item Verify OPC UA security mode configuration from a packet-level session capture.
  \item Identify event buffer overflow, exception responses, and authentication handshakes in captures.
  \item Automate protocol monitoring with tshark scripts for recurring or unattended analysis tasks.
\end{itemize}
