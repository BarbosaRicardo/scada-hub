{\small\sffamily\color{mgray}\MakeUppercase{Course Description}}
\smallskip

{\small Modbus is the most widely deployed fieldbus protocol in industrial automation, present in virtually every manufacturing and utility installation worldwide. This course covers RTU mode on RS-485, ASCII mode, and Modbus TCP over Ethernet, along with the register data model, function codes, exception handling, and data type encoding across vendor implementations. Security limitations and network-layer compensating controls are addressed in the context of modern OT network architecture.}

\smallskip
{\small\sffamily\color{mgray}\MakeUppercase{Prerequisites}}
\smallskip

{\small Basic TCP/IP networking concepts. No prior SCADA or industrial protocol experience required.}

\smallskip
{\color{mnavy}\rule{\linewidth}{1.2pt}}
\smallskip
{\small\sffamily\color{mgray}\MakeUppercase{Course Topics}}
\smallskip

\setlist[enumerate]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{mblue}{\small\bfseries\arabic*.}}
\begin{enumerate}\small
  \item \textbf{Introduction:} Protocol origins, the 1979 register model, and why the original design persists unchanged in modern field devices.
  \item \textbf{Data Model:} Coils, discrete inputs, input registers, and holding registers — table structure, data types, and access rules.
  \item \textbf{RTU Mode:} RS-485 framing, CRC-16 integrity, multi-drop bus wiring, and inter-frame silence on serial Modbus links.
  \item \textbf{ASCII Mode:} Hex-encoded framing, colon delimiter, overhead comparison to RTU, and legacy HMI equipment identification.
  \item \textbf{Modbus TCP:} Port 502, the MBAP header, transaction ID pipelining, and security implications of Ethernet transport without authentication.
  \item \textbf{Function Codes:} Read, write, and diagnostic codes; the subset covering ninety-five percent of deployments; error response handling.
  \item \textbf{Exception Codes:} Eight exception identifiers, meanings, and systematic diagnosis from exception response to root cause.
  \item \textbf{Data Types:} Register-pair conventions, byte order, word order, and floating-point encoding across vendor implementations.
  \item \textbf{Troubleshooting:} Addressing offset errors, byte-order mismatches, RS-485 termination faults, firewall drops, and poll rate overruns.
  \item \textbf{Security:} Protocol-level limitations and compensating controls: network segmentation, unidirectional gateways, and anomaly detection.
  \item \textbf{Practice Lab:} Register mapping, Exception Code 02 tracing, serial-to-TCP gateway configuration, byte-order diagnosis, and Wireshark traffic analysis.
\end{enumerate}

\smallskip
{\color{mnavy}\rule{\linewidth}{0.4pt}}
\smallskip
{\small\sffamily\color{mgray}\MakeUppercase{Learning Outcomes}}
\smallskip

\setlist[itemize]{noitemsep, topsep=2pt, leftmargin=*, label=\textcolor{mblue}{\small$\bullet$}}
\begin{itemize}\small
  \item Configure Modbus RTU and TCP device connections from device datasheet parameters.
  \item Read and interpret function code and exception code responses in a traffic capture.
  \item Apply byte-order and register-pair conventions to map floating-point and 32-bit values.
  \item Diagnose serial and TCP-layer Modbus faults using systematic isolation procedures.
  \item Specify network-layer compensating controls appropriate for a Modbus deployment.
\end{itemize}
